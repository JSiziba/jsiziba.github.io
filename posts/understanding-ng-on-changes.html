Understanding ngOnChanges

`ngOnChanges` is an important lifecycle hook in Angular components. Here's a breakdown of its functionality:

* Tracks Changes: Angular's change detection automatically triggers it whenever one or more input properties of a component change. * Input Properties: These are properties decorated with `@Input()` in your component, allowing data to flow from parent components to child components. * Reacting to Change: The `ngOnChanges` function provides a place to write code that responds specifically to changes in these input properties.

In Summary

The code shows how to use `ngOnChanges` to: 1. Monitor changes to a 'collections' input property. 2. Log the new 'collections' value to the console.

When to Use ngOnChanges

Updating internal state: Modify the component's state based on input property changes.
Triggering actions: Fetch data, re-render the UI, etc., in response to input changes.
Data Validation/Manipulation: Perform tasks when an input property changes.
